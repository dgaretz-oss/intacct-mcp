# Asynchronous API Requests

You can use synchronous and asynchronous API requests with the Sage Intacct API. The authentication and authorization process are the same in both instances. However, asynchronous requests can help with bottlenecks and long-running operations.

## Making an Async Request

Include the following header in your request:

- **Key:** `Prefer`
- **Value:** `respond-async` (regular async) or `respond-async-serialized` (serialized async)

## Receive an ACK Response

When an asynchronous request is successfully queued, the API returns a synchronous acknowledgement (ACK) response containing:

- HTTP 202 status
- `jobId` in the response body (base64url-encoded value)
- `href` for the job-status service
- A `retry-after` header (default: 2 minutes)

### Sample ACK Response

```json
{
  "ia::result": {
    "jobId": "NjQ2NTc2MzAzMNl1Ul9qVmd6M2t4M12pPdEJya2Jt65Y2dBQUFBQTE",
    "state": "queued",
    "queuedDateTime": "2025-11-11T00:47:12Z",
    "href": "/services/core/async/job-status?jobId=NjQ2NTc2MzAzMVl1Ul9qVmd6M2t4M2pPdEJya2J5Y2dBQUFBQTE"
  },
  "ia::meta": {
    "totalCount": 1
  }
}
```

## Query for Job Status

Submit a GET request using the URI from the ACK response `href`. Notes:

- The job status service can only query jobs submitted by the application in the OAuth token.
- The job status service returns job state but not the process response of the original request.
- Details for `deliveryStatus` are available for four days.

### Job State Meanings

| State | Description |
|-------|-------------|
| `queued` | Job is in the queue. |
| `inTransit` | Job is processing. |
| `delivered` | Job completed and a response was sent. |
| `dead` | Job failed or expired. |
| `cancelled` | Job was manually cancelled. |
| `deferred` | Job delayed for later processing. |

### Sample Result: Still in Queue

```json
{
  "ia::result": {
    "jobId": "NjQ2NTc2MzAzMVl1Ul9qVmd6M2t4M2pPdeJya2J5Y2dBQUFBQTE",

    "href": "/services/core/async/job-status?jobId=..."

  }
}
```

## Receive Responses

After processing, the async result is sent to the webhook specified in the request header. The result includes:

- **`X-IA-Sage-Signature`** — Signature generated by signing the response payload. Value is specified using:
  - An application secret if `X-IA-Webhook-EndpointURL` is provided.
  - A webhook secret if `X-IA-Webhook-Id` request header is provided.
- **`X-IA-Respond-Code`** — The status code of the response (should match synchronous equivalent).
- **`X-IA-Original-Request-Id`** — The ID value of the original API request.
